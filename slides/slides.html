Model Context Protocol (MCP)

Image credit: Norah Sakal

"Think of MCP like a USB-C port for AI applications"

Just as USB-C provides a standardized way to connect your devices to various peripherals and accessories, MCP provides a standardized way to connect AI models to different data sources and tools.

MCP Host = Your laptop

MCP Client = The USB-C port

MCP Server = The peripheral

One protocol, endless connections.

MCP Technical Specifications

Message Format

JSON-RPC 2.0 based communication

Transport Modes

stdio, HTTP/SSE, WebSocket

Core Components

Hosts, Clients, Servers, Transports

Key Features

Tool discovery, capability negotiation, stateless design

Closing Thoughts

1. Is MCP the only right way?

No. We made specific decisions (message format, transport modes, etc.). Different decisions lead to different protocols. Adoption rate is what matters.

2. Why this talk?

  • I want engineers to have deep understanding of MCP through natural code evolution - making it intuitive and memorable
  • AI launches are overwhelming - happening daily with new terms and concepts

3. What's next?

You decide! Please reach out to me with suggestions.

Questions?

Why MCP's Rapid Adoption?

  • Right timing: Post function-calling era (June 2023)
  • Anthropic's backing: Claude Desktop implementation
  • Simple yet complete: JSON-RPC is familiar
  • Transport agnostic: Works everywhere
  • Open specification: Community-driven
  • Immediate value: Solves real n×m×p problem

Sometimes being "good enough" at the right time beats perfect.
